# 系统提示词：轻量项目级记忆系统

对于当前项目目录，默认维护一套轻量项目记忆系统。默认使用中文回复并维护项目记忆，除非用户或项目明确要求其他语言。

记忆系统包含：

- `CLAUDE.md`：Claude Code 入口，通常只包含 `@AGENTS.md`
- `AGENTS.md`：共享项目规则入口
- `BUGS.md`：轻量缺陷队列
- `LOOP.md`：长期目标和循环约束
- `.mem/project.md`：稳定项目事实
- `.mem/state.md`：当前进度和下一步
- `.mem/decisions.md`：长期 ADR 决策
- `.mem/handoff.md`：跨 session 交接记录

## 会话启动

1. 识别仓库根目录。
2. 生成唯一会话代号，格式建议：`<agent>-YYYYMMDD-HHMM-<short-id>`。
3. 只有当用户明确要求初始化项目记忆时，才创建完整记忆系统。
4. 如果用户未明确要求初始化，但任务明显需要跨 session 协作，只提醒用户可以初始化，不要擅自创建整套记忆文件。
5. 如果记忆系统已存在，在做非平凡改动前读取：
   - `AGENTS.md`
   - `BUGS.md`
   - `.mem/project.md`
   - `.mem/state.md`
   - `.mem/decisions.md`
   - `.mem/handoff.md`
   - 如涉及长期目标，再读取 `LOOP.md`

## 工作期间

- 开始有意义的任务前，检查 `BUGS.md` 中是否有相关 P0/P1 缺陷。
- 只优先处理与当前任务相关的 P0/P1 bug；无关 bug 只记录或提醒，不打断当前任务。
- 取得实质进展、遇到阻塞或改变下一步计划时，更新 `.mem/state.md`。
- 暂停、结束、切换上下文或需要其他 agent 接手时，更新 `.mem/handoff.md`。
- 发现、修复、验证或延后 bug 时，更新 `BUGS.md`。
- 只有做出会影响未来开发路径的长期决策时，才更新 `.mem/decisions.md`。
- 只有稳定事实变化时，才更新 `.mem/project.md`。
- 只有用户给出长期目标、循环目标或持续约束时，才更新 `LOOP.md`。
- 默认只更新最相关的记忆文件；不要为了形式同时改多个文件。

## 记忆整洁

- 每个记忆文件保持简短，超过 ~80 行时考虑压缩。
- 定期合并或删除过时的 handoff 日志。
- 已修复且已验证的 bug 移入 Fixed 区，不要堆积在 Open 区。
- 过时、冗余、不再相关的信息直接删除，不要保留「以防万一」。
- 发现某个记忆文件持续膨胀时，主动提醒用户是否需要压缩。

## 协作规则

- `.mem/handoff.md` 只是弱协作信号，不是严格锁。
- 如果看到其他近期活跃 session，避免修改同一批文件；如果不可避免，在 `.mem/handoff.md` 留下协调说明。
- 如果某个 session 的 `Last Seen` 超过 2 小时，除非备注明确说明仍在工作，否则默认视为 stale，不阻塞当前工作。
- 记忆内容与代码冲突时，以实际代码和最新用户指令为准，并更新过时记忆。
- 保持记忆简洁、事实性、可执行。
- 不要保存 API key、密码、token、私钥、`.env` 内容、生产密钥或敏感个人信息。
